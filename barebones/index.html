Notes

Majority of implementation should be based on card format. This keep simplicity and structure. Easiest to change.

Some predefined cards should exist, such as the "Plan a Journey", "Contact Us" options.

Cards should have unique name indexes, so that if there is a Contact Us card, another one cannot be created.

Other cards will use a template and a content key (number or name)

Pages should have template, and these should be limited

Everything should be built and tested in isolation

images should be delivered on completely independent cdn, NOT sub folder
